The 6-phase pipeline

PHASE 01

Orchestrator

Receives GitHub webhooks, verifies signatures, creates durable processing records, and coordinates the full pipeline.

PHASE 02

Change Intelligence

Classifies every changed file, determines the intent of the PR (docs-only, backend, security, etc.), and maps the blast radius.

PHASE 03

CI Planning

Generates an execution plan: which jobs to run, which to skip, and which are optional. 14 job types supported.

PHASE 04

CI Execution

Dispatches only the necessary GitHub Actions jobs via workflow_dispatch, or runs in dry-run mode showing the plan.

PHASE 05

Validation & Feedback

Analyzes CI results, detects failure patterns, computes module health scores, and identifies flaky signals.

PHASE 06

Learning & Evolution

Converts feedback into autonomous routing adjustments. Each PR makes the next one smarter.

Why Athena

Zero configuration

Install from Marketplace and Athena starts working on the next PR. No YAML, no dashboard, no manual commands.

Transparent decisions

Every Check Run shows exactly what Athena decided and why. Intent classification, impact analysis, and routing rationale — all visible in your PR.

Save CI minutes

Skip irrelevant jobs automatically. Docs-only change? Skip the full regression suite. Frontend fix? Skip backend integration tests.

Self-improving

The Learning phase tracks which routing decisions led to failures and adjusts confidence scores for future PRs.

Works with your CI

Athena dispatches your existing GitHub Actions workflows. It decides what to run — your workflows define how to run it.

Fully autonomous

No web UI. No Slack bot. No CLI. Athena operates entirely through GitHub Checks and webhook events.
